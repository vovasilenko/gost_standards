## Скачивание стандартов ГОСТ

### Вариант 1: Через терминал (самый простой)

В Linux есть мощная утилита `wget`, которая умеет скачивать сайты целиком или только файлы определенных расширений.

Откройте терминал (Ctrl+Alt+T) и выполните следующую команду:

```bash
wget -r -l1 -nd -np -A pdf $LINK_ADDRESS
```

**Разбор параметров:**
* `-r`: рекурсивное скачивание.
* `-l1`: глубина перехода по ссылкам — 1 (только те файлы, ссылки на которые есть прямо на этой странице).
* `-nd`: (no directories) не создавать структуру папок как на сайте, свалить всё в одну текущую папку.
* `-np`: (no parent) не подниматься выше по иерархии сайта.
* `-A`: (accept) список расширений через запятую, которые нужно скачать (например, pdf, doc).

Ошибки типа `Connection refused` (соединение отклонено) обычно означают, что сервер «видит» ваш запрос, но по какой-то причине не дает программе установить связь.

Есть три основные причины:

1.  **Блокировка по User-Agent:** Сайт понимает, что к нему пришел бот (стандартный `wget`), и сразу закрывает дверь.
2.  **Протокол (HTTPS vs HTTP):** Некоторые старые сайты плохо работают с HTTPS через терминал или требуют специальных настроек SSL.
3.  **Локальный брандмауэр или прокси:** Но чаще всего это именно защита сайта от парсинга.
4. **Использование VPN:** В регионах с ограничением на VPN бывает стоит фаерволл.

**Как это исправить?**

Попробуйте «притвориться» обычным браузером и отключить проверку сертификатов. Вот обновленная команда:

```bash
wget -r -l1 -nd -np -A pdf --user-agent="Mozilla/5.0 (X11; Ubuntu; Linux x86_64; rv:109.0) Gecko/20100101 Firefox/115.0" --no-check-certificate $LINK_ADDRESS
```

**Что мы добавили:**
* `--user-agent="..."`: Теперь сайт думает, что зашел человек через Firefox на Ubuntu.
* `--no-check-certificate`: На случай, если проблема в старых или самоподписанных SSL-сертификатах сайта.
